# Security Policy

## Reporting a Vulnerability

Please report suspected security issues through the repository's private security advisory workflow if available. If private advisories are not enabled, open an issue with enough detail to reproduce the concern, but do not include secrets, credentials, private project files, or sensitive operational data.

## Scope

Security reports should focus on the BOEF application source, packaging scripts, dependency handling, project-file parsing, export behavior, and local data handling.

BOEF is a local desktop engineering application. Users should still validate downloaded releases, review dependencies, and avoid opening untrusted project files outside an appropriate local security context.
